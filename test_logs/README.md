# HiLog 日志过滤器 - 测试指南

## 📁 测试文件说明

### 新创建的测试文件

#### 1. large_volume.txt (50条日志)
- **内容**: 完整的应用运行日志
- **时间范围**: 01-15 08:00:00 - 09:20:00
- **包含**: 各种日志级别 (V/D/I/W/E)
- **场景**: 系统启动、用户操作、数据库操作、支付流程等

#### 2. multi_time_range.txt (60条日志)
- **内容**: 跨多天的日志，测试时间过滤功能
- **时间范围**: 01-14 23:55:00 - 01-16 14:45:08
- **包含**: 系统重启、定时任务、全时段活动
- **场景**: 完整的24小时周期 + 跨天场景

#### 3. error_logs.txt (27条日志)
- **内容**: 各种错误和警告日志
- **时间范围**: 01-16 10:00:00 - 10:50:10
- **包含**: 大量 Error 和 Fatal 级别日志
- **场景**: 支付失败、数据库错误、网络错误、磁盘满等

## 🧪 功能测试步骤

### 测试 1: 文件选择功能

**步骤**:
1. 点击 "选择单个文件" 按钮
2. 导航到 `test_logs` 目录
3. 选择 `large_volume.txt`
4. ✅ 验证: 文件出现在文件列表中

5. 点击 "选择多个文件" 按钮
6. 选择 `multi_time_range.txt` 和 `error_logs.txt`
7. ✅ 验证: 两个文件都出现在列表中

8. 点击 "选择文件夹" 按钮
9. 选择 `test_logs` 文件夹
10. ✅ 验证: 所有 .txt 文件都被加载

### 测试 2: 时间过滤功能

**步骤**:
1. 选择 `multi_time_range.txt` 文件
2. 设置起始时间: `01-16 08:00:00`
3. 点击 "当前时间" 按钮设置当前时间
4. ✅ 验证: 时间预览显示正确

### 测试 3: 开始过滤

**步骤**:
1. 确保已选择文件和设置时间
2. 点击 "开始过滤" 按钮
3. ✅ 验证:
   - 显示加载状态
   - 统计信息更新
   - 日志列表显示过滤结果

### 测试 4: 日志列表功能

**步骤**:
1. 在日志列表中滚动
2. ✅ 验证: 虚拟滚动流畅，无卡顿
3. 点击任意日志行
4. ✅ 验证: 弹出详情对话框

### 测试 5: 日志详情功能

**步骤**:
1. 查看日志详情弹窗
2. ✅ 验证: 显示所有字段（时间戳、等级、PID/TID、Tag、内容等）
3. 点击 "复制" 按钮
4. ✅ 验证: 提示已复制，内容格式正确
5. 点击 "复制原始日志" 按钮
6. ✅ 验证: 原始日志被复制

### 测试 6: 统计信息功能

**步骤**:
1. 查看左侧统计面板
2. ✅ 验证:
   - 总日志数正确
   - 过滤后数量正确
   - 占比计算正确
   - 等级分布显示

### 测试 7: 导出功能

**步骤**:
1. 点击 "导出" 按钮
2. 选择保存位置
3. ✅ 验证: 成功导出 TXT 文件

4. 点击 "更多格式" → "导出为JSON"
5. ✅ 验证: 成功导出 JSON 文件

6. 点击 "更多格式" → "导出为CSV"
7. ✅ 验证: 成功导出 CSV 文件

### 测试 8: 清空功能

**步骤**:
1. 点击 "清空" 按钮
2. ✅ 验证: 所有日志被清空
3. 再次点击文件选择器的 "清空所有文件"
4. ✅ 验证: 文件列表被清空

## 📊 推荐测试场景

### 场景 1: 大量日志测试
```
文件: multi_time_range.txt
时间: 01-15 00:00:00
预期: 显示所有01-15的日志
```

### 场景 2: 精确时间过滤
```
文件: multi_time_range.txt
时间: 01-16 14:42:52
预期: 只显示14:42:52之后的日志（主要是商城日志）
```

### 场景 3: 错误日志分析
```
文件: error_logs.txt
时间: 01-16 10:00:00
预期: 显示所有错误日志，统计中Error占比较高
```

```
文件: multi_time_range.txt
时间: 01-15 12:00:00
预期: 显示从01-15中午到01-16的日志
```
