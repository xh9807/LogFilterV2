## 核心技术要点

### 1. Electron架构
```
┌─────────────────────────────────┐
│   Main Process (Node.js)        │
│   - 文件系统操作                 │
│   - IPC通信                     │
│   - 窗口管理                     │
└─────────────────────────────────┘
            ↕ IPC
┌─────────────────────────────────┐
│   Renderer Process (React)      │
│   - UI渲染                      │
│   - 用户交互                     │
│   - 状态管理                     │
└─────────────────────────────────┘
```

### 2. HiLog日志格式
```
01-16 14:42:52.788 62125 62125 I A03D00/com.jd.hm.mall/JSAPP: [[1,"banneractiveIndex",1.0]]
└─┬─┘ └───┬───┘ └─┬─┘ └─┬─┘ └─┬─┘ └─────────────┬─────────────┘ └───────┬───────┘
 日期      时间    PID   TID  等级       Tag                   Content
```

### 3. 性能优化策略
- **流式读取**: 使用Node.js Stream避免内存溢出
- **虚拟滚动**: react-window实现10万+条日志流畅滚动
- **Web Worker**: 在Worker线程中解析日志，避免阻塞UI
- **批量IPC**: 减少IPC通信次数，批量传输数据

## 开发注意事项

1. **文件大小限制**: 严格检查100MB限制，超过则跳过并提示用户
2. **流式读取**: 大文件必须使用Stream，不可一次性加载到内存
3. **时间精度**: 支持毫秒级时间比较
4. **日志格式**: 严格按照HiLog格式解析，容错处理异常行
5. **性能优先**: 使用虚拟滚动、Web Worker等优化手段
6. **用户体验**: 添加进度提示和错误反馈
7. **类型安全**: 充分利用TypeScript类型系统
8. **IPC通信**: 主进程和渲染进程通信要注意安全性
