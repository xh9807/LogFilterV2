# 鸿蒙HiLog日志过滤系统 - 软件需求设计书

## 1. 项目概述

### 1.1 项目名称
鸿蒙HiLog日志过滤系统 (LogFilterV2)

### 1.2 项目背景
在鸿蒙应用开发和调试过程中，开发者需要从大量的HiLog日志文件中快速定位特定时间段的日志信息。本系统是一个基于Electron + React的桌面应用，旨在提供一个便捷的跨平台日志分析工具。

### 1.3 项目目标
- 提供跨平台的桌面日志分析工具（Windows/macOS/Linux）
- 支持大文件日志的读取（限制100MB以内）
- 提供精确的时间过滤功能
- 解析HiLog日志格式并提取关键信息
- 展示过滤后的日志结果

### 1.4 目标用户
- 鸿蒙应用开发工程师
- 测试工程师
- 技术支持人员

## 2. 功能需求

### 2.1 文件选择模块

#### 功能描述
用户可以选择包含日志文件的目录或单个文件，系统自动扫描并加载符合条件的日志文件。

#### 详细需求
1. **目录/文件选择**
   - 提供目录选择按钮（选择整个文件夹）
   - 提供文件选择按钮（选择单个或多个文件）
   - 使用原生文件选择对话框
   - 支持拖拽文件/文件夹到应用窗口

2. **文件扫描与过滤**
   - 扫描选定目录下的所有文件
   - 仅支持 `.txt` 和 `.json` 格式的文件
   - 递归扫描子目录（可配置）

3. **文件大小限制**
   - 检查每个文件的大小
   - 超过100MB的文件不予读取
   - 给用户明确提示哪些文件因过大被跳过

4. **大文件读取优化**
   - 使用流式读取（Node.js Stream）
   - 分批读取避免阻塞主线程
   - 使用Web Worker处理大文件解析
   - 实时显示读取进度

### 2.2 时间选择模块

#### 功能描述
提供友好的UI界面让用户选择日志的起始时间。

#### 详细需求
1. **时间选择器组件**
   - 使用Ant Design或Material-UI的DatePicker/TimePicker
   - 或自定义下拉选择器（月、日、时、分、秒）
   - 格式：MM-dd HH:mm:ss

2. **时间范围**
   - 月：01-12
   - 日：01-31
   - 时：00-23
   - 分：00-59
   - 秒：00-59

3. **时间验证**
   - 验证日期的合法性
   - 提供快捷选项（今天、昨天、当前时间）
   - 支持时间清空（表示不过滤时间）

4. **用户体验**
   - 支持手动输入时间
   - 实时预览选择的时间
   - 提供时间格式说明

### 2.3 日志解析模块

#### 功能描述
解析HiLog日志格式，提取时间戳、日志等级等关键信息。

#### 详细需求
1. **日志格式识别**
   - 支持标准HiLog日志格式
   - 格式：`MM-dd HH:mm:ss.SSS PID TID Level Tag/Message: Content`

2. **字段解析**
   ```typescript
   示例：01-16 14:42:52.788 62125 62125 I A03D00/com.jd.hm.mall/JSAPP: [[1,"banneractiveIndex",1.0]]

   解析字段：
   - 时间戳：01-16 14:42:52.788 (MM-dd HH:mm:ss.SSS)
   - PID：62125
   - TID：62125
   - 等级：I (Info)
   - Tag：A03D00/com.jd.hm.mall/JSAPP
   - 内容：[[1,"banneractiveIndex",1.0]]
   ```

3. **日志等级映射**
   ```
   V - Verbose (灰色)
   D - Debug (绿色)
   I - Info (蓝色)
   W - Warn (橙色)
   E - Error (红色)
   F - Fatal (深红色)
   ```

4. **容错处理**
   - 无法解析的行保留原始内容
   - 标记为"解析失败"
   - 统计解析失败的行数

### 2.4 日志过滤模块

#### 功能描述
根据用户选择的时间、等级、Tag等条件过滤日志。

#### 详细需求
1. **时间过滤**
   - 将用户选择的时间与日志时间戳比较
   - 支持毫秒级精度
   - 过滤规则：日志时间 >= 起始时间
   - 可选：结束时间过滤

2. **多维度过滤**（可选增强功能）
   - 按日志等级过滤（只看Error/Warn等）
   - 按Tag过滤（支持模糊匹配）
   - 按PID/TID过滤
   - 按关键词搜索日志内容

3. **性能优化**
   - 使用高效的字符串匹配算法
   - 避免重复解析
   - 支持取消过滤操作
   - 使用Web Worker避免阻塞UI

### 2.5 结果展示模块

#### 功能描述
以清晰的方式展示过滤后的日志。

#### 详细需求
1. **列表展示**
   - 使用虚拟滚动列表（react-window或react-virtualized）
   - 支持数千条日志流畅滚动
   - 每条日志显示：时间、等级、Tag、内容摘要

2. **日志高亮**
   - 根据日志等级使用不同颜色
   - 关键词搜索高亮
   - 当前行高亮

3. **详细信息**
   - 点击展开查看完整日志内容
   - 显示所有字段（时间戳、PID、TID、等级、Tag、内容）
   - 支持复制单条日志

4. **统计信息**
   - 显示总日志数、过滤后日志数
   - 各等级日志数量统计
   - 文件信息（文件名、大小、行数）

5. **导出功能**
   - 导出过滤后的日志为txt文件
   - 导出为JSON格式（包含所有字段）
   - 支持复制到剪贴板

### 2.6 附加功能（可选）

1. **搜索功能**
   - 全文搜索日志内容
   - 支持正则表达式
   - 高亮搜索结果

2. **书签功能**
   - 标记重要的日志行
   - 快速跳转到书签位置

3. **日志对比**
   - 对比两个时间段的日志差异

4. **主题切换**
   - 明色/暗色主题

## 3. 非功能需求

### 3.1 性能要求
- 启动时间 < 3秒
- 100MB文件读取和解析 < 10秒
- UI响应时间 < 100ms
- 支持同时处理10+个文件
- 虚拟滚动支持10万+条日志

### 3.2 易用性要求
- 界面简洁直观，符合桌面应用规范
- 支持拖拽文件
- 提供操作提示和错误反馈
- 支持快捷键操作
- 提供使用说明

### 3.3 稳定性要求
- 异常文件不导致应用崩溃
- 内存占用 < 500MB
- 处理超大文件时不卡死
- 自动保存用户配置

### 3.4 兼容性要求
- Windows 10/11
- macOS 10.15+
- Linux (Ubuntu 20.04+)

## 5. 数据模型

### 5.1 日志条目模型
```typescript
interface LogEntry {
  // 原始数据
  rawLine: string;           // 原始日志行
  lineNumber: number;        // 行号（在文件中的位置）

  // 时间信息
  timestamp: string;         // "01-16 14:42:52.788"
  date: string;              // "01-16"
  time: string;              // "14:42:52.788"
  month: number;             // 1-12
  day: number;               // 1-31
  hours: number;             // 0-23
  minutes: number;           // 0-59
  seconds: number;           // 0-59
  milliseconds: number;      // 0-999

  // 进程信息
  pid: number;               // 进程ID
  tid: number;               // 线程ID

  // 日志信息
  level: LogLevel;           // 日志等级
  tag: string;               // 标签
  content: string;           // 日志内容

  // 元数据
  sourceFile: string;        // 来源文件路径
  parseError?: string;       // 解析错误信息（如果有）
}

enum LogLevel {
  VERBOSE = 'V',
  DEBUG = 'D',
  INFO = 'I',
  WARN = 'W',
  ERROR = 'E',
  FATAL = 'F'
}

// 日志等级配置
interface LogLevelConfig {
  level: LogLevel;
  label: string;
  color: string;             // 前景色
  backgroundColor: string;   // 背景色
  icon?: string;             // 图标
}
```

### 5.2 过滤配置模型
```typescript
interface FilterConfig {
  // 时间过滤
  startTime?: LogTime;       // 起始时间
  endTime?: LogTime;         // 结束时间（可选）

  // 等级过滤（可选）
  levels?: LogLevel[];       // 选中的日志等级

  // Tag过滤（可选）
  tags?: string[];           // 选中的Tags

  // 关键词搜索（可选）
  keyword?: string;          // 搜索关键词
  useRegex?: boolean;        // 是否使用正则
}

interface LogTime {
  month: number;
  day: number;
  hours: number;
  minutes: number;
  seconds: number;
  milliseconds?: number;
}
```

### 5.3 文件信息模型
```typescript
interface FileInfo {
  path: string;              // 文件完整路径
```

## 6. 用户交互流程

```
1. 用户启动应用
   ↓
2. 界面显示欢迎信息或操作提示
   ↓
3. 用户选择日志文件/文件夹
   方式1: 点击"选择文件"按钮
   方式2: 点击"选择文件夹"按钮
   方式3: 拖拽文件/文件夹到窗口
   ↓
4. 系统扫描并验证文件
   - 检查文件格式（txt/json）
   - 检查文件大小（<100MB）
   - 显示找到的文件列表
   ↓
5. 用户设置过滤条件
   - 选择起始时间（必须）
   - 选择结束时间（可选）
   - 选择日志等级（可选）
   - 输入搜索关键词（可选）
   ↓
6. 用户点击"开始过滤"按钮
   ↓
7. 系统处理日志
   - 读取文件（显示进度）
   - 解析日志格式
   - 应用过滤条件
   ↓
8. 展示过滤结果
   - 显示日志列表
   - 显示统计信息
   - 支持滚动浏览
   ↓
9. 用户可以进行的操作
   - 点击日志查看详情
   - 复制单条日志
   - 搜索关键词
   - 导出结果
   - 重新过滤
```

## 10. 风险与挑战

### 10.1 技术风险
| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 大文件读取内存溢出 | 高 | 使用流式读取 + Web Worker |
| 虚拟滚动性能问题 | 中 | 使用成熟的react-window库 |
| 日志格式解析错误 | 中 | 容错处理 + 记录错误日志 |
| IPC通信性能瓶颈 | 低 | 批量传输数据 |

### 10.2 业务风险
| 风险 | 影响 | 应对方案 |
|------|------|----------|
| HiLog格式变化 | 中 | 可配置的解析规则 |
| 用户需求变更 | 低 | 模块化设计，便于扩展 |

### 10.3 用户体验风险
| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 界面复杂难用 | 中 | 遵循桌面应用设计规范 |
| 操作响应慢 | 高 | 使用Web Worker + 进度提示 |
| 缺少操作引导 | 低 | 添加使用说明和提示 |
